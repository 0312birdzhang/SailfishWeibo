认证说明

以下是关于微博登陆/认证的一些说明

API Key认证

API Key认证，也就是微博第三方授权登陆认证，是微博官方推介的一种第三方登陆认证方式，也是相当安全的认证方式。使用这种认证方式，会打开一个浏览器页面，在页面中用户输入帐号密码并且同意相关的授权请求即可以使用微博相关功能。由于这种认证方式需要向新浪申请第三方登陆授权，并且个人用户申请的登陆授权限制了大量的功能，所以，目前SailfishWeibo使用了“黑魔法”登陆方式。具体何为“黑魔法”，相信使用这个微博的用户也都知道，就不再细说了。

用户帐号密码登陆

这种方式使用了模拟网页端登陆的方式，也就是说，用户需要在SailfishWeibo中输入自己的微博帐号/密码，SailfishWeibo通过一些特殊的方式，模拟网页登陆的方式，最终登陆用户的微博。由于是模拟网页登陆的方式，所以使用此种方式可以得到几乎所有的微博功能。由于需要输入帐号密码，考虑到安全性SailfishWeibo不会在本地保存用户的帐号密码，只会保留最终登陆成功后的私人密钥。帐号密码登陆方式使用了单独的安装包实现，使用此种方式，需要单独安装登陆模块支持包。
单独安装包下载地址: 从Openrepos下载     从Github下载    

Access Token

AccessToken和userID是使用微博api必须传入的参数，考虑到方便性，AccessToken和userID是被明文保存在本地空间的，由于明文本地保存，所以存在一定的泄露可能性，如果用户担心泄密，可以在每次使用微博后手动注销登陆。

隐私说明

SailfishWeibo微博本身不会使用/泄露任何用户的数据，也不会上传这些数据到除新浪微博系统本身外的任何其他第三方。但是考虑到登陆方式/AccessToken和userID保存等相关的方面，仍然存在泄露用户相关隐私的风险，开发者对此不承担任何责任。
